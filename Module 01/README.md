## Overview

This module introduces the foundational argument of the course: that studying the biological brain can expand our understanding of intelligence and inspire better AI design. Rather than asking how to scale existing architectures, Module 1 asks what design principles evolution discovered that modern AI has overlooked.

Key themes covered:
- The hidden assumptions baked into modern AI (discrete computation, static memory, dense processing, separate training/deployment)
- The brain's 20-watt energy budget as a design constraint, not a footnote
- Dense vs. sparse computation and why the distinction matters
- A brief history of how AI and neuroscience grew apart — and why they're reconnecting
- Why modern innovations (RAG, agents, reasoning models, chatbot memory) add scaffolding but don't change fundamental architecture
- Memory as the core of intelligence: static vs. dynamic, retrieval vs. reconstruction

## Key Concepts (Glossary Preview)

| Term | Brief Definition |
|---|---|
| Dense Computation | Most processing units active on every pass; standard in neural networks |
| Sparse Computation | Only a small fraction of units active at any moment; the brain's default |
| Static Memory | Weights or storage fixed after training; doesn't change during use |
| Dynamic Memory | Memory that is reconstructed and modified through use; biological default |
| Scaffolding | External systems (RAG, memory stores, tool use) added to a static model |
| Catastrophic Forgetting | When learning new information overwrites previously learned knowledge |
| In-Context Learning | Adapting behavior via prompt examples without changing model weights |
